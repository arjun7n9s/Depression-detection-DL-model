02

Data Pipeline

Dataset Audit & Manifest Generator

New step before training. Extracts sequences, validates files, catches length mismatches (e.g. 14 mismatched subjects in D-vlog). Compiles all windows into a singular `manifest.jsonl`.

E-DAIC Feature Extraction & Processing

  • Batch-extract `.tar.gz` and parse files automatically.
  • Visual: 6 pose + 8 gaze + 17 AU_r = 31 dims
  • Acoustic: 23 valid features after metadata stripping
  • Drop frames where OpenFace confidence < 50%

Unified Model Input Format

{
    "visual":         (num_windows, 30, visual_dim),
    "acoustic":       (num_windows, 30, acoustic_dim),
    "label_binary":   int,           # 0 or 1
    "label_phq":      float,         # PHQ score or NaN
    "subject_id":     str,
    "dataset_source": str,           # "edaic" or "dvlog"
    "quality_flags":  list[str],
    "modality_mask":  dict,
}

Training Strategy

Benchmark Ladder Strategy

  Level 1  В·  Acoustic-only baseline (each dataset separately)
  Level 2  В·  Visual-only baseline (each dataset separately)
  Level 3  В·  Acoustic + Visual bimodal
  Level 4  В·  Cross-dataset pretraining (D-vlog to E-DAIC)
  Level 5  В·  Full multi-task multimodal

  Rule: Each level must beat the previous. If Level 3 does not 
  beat Level 1/2, then the fusion model is broken.

Real-Time Inference

This is a prototype behavioral screening system, not a clinical diagnosis software. The inference will make this explicitly known.

Feature Space Bridge

Since the offline model uses OpenFace AUs and the live model uses MediaPipe FaceMesh, a Feature Space Bridge projection will be trained instead of inventing hand-crafted AU mappings.
